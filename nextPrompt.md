## Промпт: Помощь с доработкой race_test.sh

### Контекст
Делаю тестовое задание "Заявки в ремонтную службу" на Laravel. Нужно доделать bash-скрипт `race_test.sh` для проверки защиты от гонок.

### Что уже работает в проекте
- Локальный сервер: http://localhost:8080
- Панель диспетчера: `/dispatcher/requests`
- Панель мастера: `/master/requests`
- Тестовые пользователи:
  - Диспетчер: dispatcher@example.com / password
  - Мастер Иван: ivan@example.com / password
  - Мастер Петр: petr@example.com / password

### Особенности
- CSRF токен обязателен для POST/PATCH запросов
- В HTML страницы диспетчера ID заявок выглядят так: 
  `<form method="POST" action="http://localhost:8080/dispatcher/requests/15/assign">`
- Нужно получать свежий CSRF перед каждым запросом (через /login)

### Что должен делать скрипт
1. Автоматически логиниться как диспетчер
2. Создавать тестовую заявку (POST /requests)
3. Находить ID созданной заявки из HTML страницы диспетчера
4. Назначать на эту заявку мастера (PATCH /dispatcher/requests/{id}/assign)
5. Логиниться как этот мастер
6. Запускать 5 параллельных PATCH-запросов на `/master/requests/{id}/take`
7. Проверять что только 1 успешен (HTTP 302), остальные 4 получают отказ
